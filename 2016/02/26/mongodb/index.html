mongodb

标签（空格分隔）： mongodb

#创建数据库
cd 进入bin
./mongo –启动数据库
use （数据库名） –创建一个名为xxx的数据库
db –检查当前选定的数据库
show dbs –检查数据列表

#删除数据库
db.dropDatabase() –删除数据库

#创建集合
db.createCollection(name,options) –options可选
show collections –查看创建的集合

#删除集合
db.(集合名).drop()

#插入文档
要是想将数据写入mongodb集合中，需要使用inset()和save()方法

–inset()基本格式
db.COLLECTION_NAME().insert(document)

范例1

db.mycol.insert({
　　“_id”: 1,
　　“title”: ‘MongoDB Overview’,
　　“description”: ‘MongoDB is no sql database’,
　　“by”: ‘tutorials point’,
　　“url”: ‘http://www.tutorialspoint.com‘,
　　“tags”: [‘mongodb’, ‘database’, ‘NoSQL’],
　　“likes”: 100
})

##查询文档
查询数据，用find()
–db.COLLECTION_NAME.find()
–pretty() 格式化显示结果
–find() 非机构化显示所有文档

db.mycol.find().pretty()

##MongoDB 中的 And条件
在 find() 方法中，如果传入多个键，并用逗号(,)分隔它们，那么 MongoDB 会把它看成是 AND 条件。
–db.mycol.find({key1:value1, key2:value2}).pretty()

db.mycol.find({“by”:”tutorials point”,”title”: “MongoDB Overview”}).pretty()
{
　　“_id”: ObjectId(7df78ad8902c),
　　“title”: “MongoDB Overview”,
　　“description”: “MongoDB is no sql database”,
　　“by”: “tutorials point”,
　　“url”: “http://www.tutorialspoint.com“,
　　“tags”: [“mongodb”, “database”, “NoSQL”],
　　“likes”: “100”
}

##MongoDB 中的 OR 条件
若基于 OR 条件来查询文档，可以使用关键字 $or。 OR 条件的基本语法格式为：

db.mycol.find(
　　{
　　　　$or: [
　　　　　{key1: value1}, {key2:value2}
　　　　]
　　}
).pretty()

##范例
db.mycol.find({$or:[{“by”:”tutorials point”},{“title”: “MongoDB Overview”}]}).pretty()
{
　　“_id”: ObjectId(7df78ad8902c),
　　“title”: “MongoDB Overview”,
　　“description”: “MongoDB is no sql database”,
　　“by”: “tutorials point”,
　　“url”: “http://www.tutorialspoint.com“,
　　“tags”: [“mongodb”, “database”, “NoSQL”],
　　“likes”: “100”
}

##结合使用ＡＮＤ和ＯＲ条件
–下例所展示文档的条件为：喜欢数大于 100，标题是 “MongoDB Overview”，或者是由 “tutorials point” 所发表的
db.mycol.find({“likes”: {&gt:10},$or: [{“by”: “tutorials point”},{“title”: “MongoDB Overview”}]}).pretty()
{
　　“_id”: ObjectId(7df78ad8902c),
　　“title”: “MongoDB Overview”,
　　“description”: “MongoDB is no sql database”,
　　“by”: “tutorials point”,
　　“url”: “http://www.tutorialspoint.com“,
　　“tags”: [“mongodb”, “database”, “NoSQL”],
　　“likes”: “100”
}
